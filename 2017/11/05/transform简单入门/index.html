transform简单入门

发表于 2017-11-05

简单介绍

transform可以用来做dom变换

实践

多个transform顺序

大家都知道transform可以一起做很多的变换操作，但是这里要清楚的一点是所有的变换操作都是根据上一次变换后的dom来做出改变。

比如我们有这样一个盒子，我们想让这个盒子往右边移动100px并且旋转45deg，这个时候我们写的代码可能是这样的：
transform: translateX(100px) rotate(45deg);
效果：

但是如果我们尝试换一种方式：
transform: rotate(45deg) translateX(100px);
这个时候我们看到的效果可能是：

很奇怪是吗？
原因是transform的dom变换其中有一个坐标轴的概念，也就是xyz轴，transform根据这三个轴在空间中的不同位置进行变换，而且上面已经讲过，transform会根据上一次这xyz轴进行操作。

第一个效果首先移动了100px但是坐标轴还是原始的坐标找，所以之后旋转的45deg我们可以得到我们想要的效果。
然而第二个效果，我们可以理解为坐标轴首先被rotate变换了45deg然后再根据x轴进行位移，变换后的坐标轴大概是这样的

最终我们可以得到的一个结论是，transform的每一次变换都进行了以下操作：
1.基于三轴进行变换
2.输出变换后的三轴
3.进行下次变换

理解了这个简单的概念，我们在实际场景中应用这个属性做一些变换操作就变得更加清晰了。

性能问题

性能问题其实不是transform引起的，而是transition导致的，在这里提一下，是因为我们利用transform除了一般需要变换dom以外可能还会结合transition，animation之类的样式做动画效果。

举一个很简单的例子，飘落物效果。
首先我们最简单的解决方案之一就是利用dom在顶部排列好比如10-20个飘落物，然后position:fixed这些dom，之后我们利用transition设置top属性来控制这些飘落物或者是利用animation做成动效。
但是这里有一个概念需要大家明白的是如果你用了top，那么在你不断改变top的时候其实是浏览器内核对dom进行了重绘，就像有些同学玩过flash的帧动画，为什么我们的gif图这么大，就是因为它也是一个帧动画，用一帧一帧的移动来形成动画效果，这无疑有大量的资源浪费。
但是如果我们不用transition: top 1s;我们改用transition: transform 1s; 然后transform: translateY(2000px);这样会也可以做我们想要的效果，这种动画效果其实就是我们flash里面的补间动画，就是浏览器自动帮我们设置好中间的动画效果，我们只要设置好开始和结束就可以了。
